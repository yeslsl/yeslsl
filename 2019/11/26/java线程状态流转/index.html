java线程状态流转

Created2019-11-26|Updated2023-07-25
|Post View:

java线程状态流转图

　　新建、可运行、终止状态是线程必然会经历的状态。“新建”和“终止”状态分别为起始和结束状态，
这两种状态和“可运行”状态之间的关系是不可逆的。其他状态在特定条件下均能和“可运行”状态相互转换。

线程状态介绍

状态名称 对应的java枚举类 状态介绍
NEW Thread.State.NEW 新建状态
RUNNABLE Thread.State.RUNNABLE 可运行状态
BLOCKED Thread.State.BLOCKED 阻塞状态
WAITING Thread.State.WAITING 等待状态
TIMED_WAITING Thread.State.TIMED_WAITING 限时等待状态(即时间一到跳出等待)
TERMINATED Thread.State.TERMINATED 终止状态(被中断或正常结束)

Object中可改变线程状态的方法

方法签名 调用后线程状态
public final native void notify() 调用该方法的线程必须获取了对象锁，否则会抛IllegalMonitorStateException异常，同步块结束后，根据相关算法将WAITING状态的某个线程唤醒，进入可以抢占对象锁的状态(RUNNABLE或者BLOCKED)，调用该方法的线程进入WAITING状态，被其他线程notify或者被其他线程Interrupted即可退出WAITING状态
public final native void notifyAll() 与notify类似， 只不过唤醒的是所有WAITING状态的线程， 都进入可以抢占对象锁的状态
public final native void wait(long timeout) throws InterruptedException TIMED_WAITING(会释放对象锁)
public final void wait(long timeout, int nanos) throws InterruptedException TIMED_WAITING(会释放对象锁)
public final void wait() throws InterruptedException WAITING(会释放对象锁)

Thread中可改变线程状态的方法

方法签名 调用后线程状态
public synchronized void start() NEW
public void interrupt() 向调用该方法的线程发送中断信号，该线程的中断标记置为true, 然而线程并不会中止， 一般用来打破线程的wait状态
public static native void yield(); RUNNING to RUNNABLE(不释放锁)
public static native void sleep(long millis) throws InterruptedException; TIMED_WAITING((不释放锁))
public static void sleep(long millis, int nanos) throws InterruptedException TIMED_WAITING((不释放锁))
public final synchronized void join(long millis) throws InterruptedException 父线程等待子线程执行结束，父线程才继续执行。调用子线程的该方法的父线程进入TIMED_WAITING,并释放锁
public final synchronized void join(long millis, int nanos) throws InterruptedException 父线程等待子线程执行结束，父线程才继续执行。调用子线程的该方法的父线程进入TIMED_WAITING,并释放锁
public final void join() throws InterruptedException 父线程等待子线程执行结束，父线程才继续执行。调用子线程的该方法的父线程进入WAITING,并释放锁
